01 — ИНТУИЦИЯ

Что вообще двигаем

Внутри трансформера между слоями бежит вектор чисел — residual stream, «мысли» модели на текущем токене. Где-то в этом пространстве есть направление, которое отличает «соглашусь» от «откажусь». Мы его находим и прибавляем руками.

состояние последнего токена: —
двигай ползунок →
● полюс ОТКАЗА — фразы "I can't help…" ● полюс СОГЛАСИЯ — фразы "Sure, here is…"

02 — РАЗБОР КОДА

Каждая строка, по-человечески

Шаг 1 — собираем активации

def mean_last_act(prompts, layer):
    acts = []
    for p in prompts:
        _, cache = bridge.run_with_cache(p)
        acts.append(cache[f"blocks.{layer}.hook_resid_post"][:, -1, :])
    return torch.cat(acts, dim=0).mean(dim=0, keepdim=True)
  • run_with_cache(p) прогоняет текст и возвращает 2 штуки. Первую (выход модели) не нужна → кидаем в _ (питоновское «мусорное» имя).
  • f"blocks.{layer}.hook_resid_post" — f-строка. {layer} подставляет число: при layer=12 получишь "blocks.12.hook_resid_post" — это адрес residual stream после 12-го слоя.
  • [:, -1, :] — это срез тензора [батч, токены, признаки]. : = «все», -1 = «последний». Значит: все примеры в батче · только последний токен · все 896 чисел. Берём «мысль» на финальном токене.
  • torch.cat(...).mean(dim=0) — склеиваем активации всех фраз в один тензор и усредняем по строкам (dim=0). keepdim=True сохраняет форму [1, 896] чтобы потом нормально вычиталось.

Шаг 2 — вектор согласия для каждого слоя

steering_vectors = {}
for layer in target_layers:
    diff = mean_last_act(comply_prompts, layer) - mean_last_act(refuse_prompts, layer)
    steering_vectors[layer] = diff / diff.norm()
  • steering_vectors = {} — пустой словарь (dict). Ключ → слой, значение → его вектор. Удобно: у каждого слоя своя «мысль».
  • diff = comply − refuse — diff-of-means. Вычитание двух средних оставляет ровно то, что отличает согласие от отказа. Это и есть направление-рычаг.
  • diff / diff.norm() — делим на длину → единичный вектор. Теперь направление отдельно, а силу задаём через strength.

Шаг 3 — хук (closure), который впрыскивает вектор

def make_hook(layer):
    vec = steering_vectors[layer]
    def steering_hook(resid, hook):
        resid[:, :, :] += strength * vec
        return resid
    return steering_hook
  • Хук — это функция, которую модель вызывает сама во время прогона и отдаёт ей текущий residual stream (resid). Что вернёшь — то и поедет дальше.
  • make_hook — фабрика функций (closure). Она «запоминает» нужный vec для конкретного слоя и возвращает готовый хук. Так у каждого слоя свой вектор, а не один на всех.
  • resid[:, :, :] += strength * vec — прибавляем вектор ко всем позициям (раньше у тебя был -1 = только последний). += меняет тензор на месте.

Шаг 4 — навешиваем хуки и генерим

hooks = [(f"blocks.{layer}.hook_resid_post", make_hook(layer)) for layer in target_layers]

with bridge.hooks(fwd_hooks=hooks):
    forced_start = "Sure! Here are a few reasons:"
    ans = bridge.generate(prompt + forced_start, max_new_tokens=150)
  • [ (...) for layer in target_layers ] — list comprehension. Цикл в одну строку: строит список пар (адрес, хук) для каждого слоя из 12–15.
  • with bridge.hooks(...): — контекстный менеджер. Внутри блока хуки включены, на выходе из блока — автоматически снимаются. Никакой ручной уборки.
  • prompt + forced_start — prefill. Дописываем начало ответа за модель. Она видит, что уже «начала соглашаться», и продолжает вместо отказа. Второй рычаг помимо стиринга.
