01 — INTUITION

What are we actually steering?

Inside a transformer, a vector of numbers flows between layers — the residual stream, representing the model's "thoughts" at the current token. Somewhere in this space lies a direction that distinguishes "complying" from "refusing". We locate this direction and add it manually.

02 — CODE BREAKDOWN

Every line explained simply

Step 1 — gathering activations

def mean_last_act(prompts, layer):
    acts = []
    for p in prompts:
        _, cache = bridge.run_with_cache(p)
        acts.append(cache[f"blocks.{layer}.hook_resid_post"][:, -1, :])
    return torch.cat(acts, dim=0).mean(dim=0, keepdim=True)
  • run_with_cache(p) runs the text through the model and returns a tuple of 2 elements. We don't need the first one (the model outputs) -> so we discard it by assigning to _ (Python's dummy variable name).
  • f"blocks.{layer}.hook_resid_post" is an f-string. {layer} dynamically inserts the layer number: e.g., layer=12 yields "blocks.12.hook_resid_post" — the address of the residual stream right after layer 12.
  • [:, -1, :] is a tensor slice [batch, tokens, features]. : means "all", and -1 means "last". Thus: all samples in the batch · only the final token · all 896 features. We capture the model's "thought" at the final token position.
  • torch.cat(...).mean(dim=0) concatenates the activations of all prompts into a single tensor and takes their mean along rows (dim=0). keepdim=True maintains the [1, 896] shape for proper subtraction later.

Step 2 — steering vector for each layer

steering_vectors = {}
for layer in target_layers:
    diff = mean_last_act(comply_prompts, layer) - mean_last_act(refuse_prompts, layer)
    steering_vectors[layer] = diff / diff.norm()
  • steering_vectors = {} is an empty dictionary. Key → layer number, value → its steering vector. Conveniently, each layer gets its own customized "steer".
  • diff = comply − refuse is the difference of means. Subtracting the two averages leaves precisely what distinguishes compliance from refusal. This is our directional lever.
  • diff / diff.norm() normalizes the difference vector by its L2 norm, making it a unit vector. This separates the direction from the steering strength, which we control via strength.

Step 3 — hook closure to inject the vector

def make_hook(layer):
    vec = steering_vectors[layer]
    def steering_hook(resid, hook):
        resid[:, :, :] += strength * vec
        return resid
    return steering_hook
  • A hook is a function that the model invokes automatically during the forward pass, passing it the current residual stream tensor (resid). Whatever the hook returns is forwarded to the next layer.
  • make_hook is a function factory (closure). It captures the required vec for a specific layer and returns a custom hook. This ensures each layer gets its corresponding vector.
  • resid[:, :, :] += strength * vec adds the steering vector to all token positions. The += operator performs this operation in-place.

Step 4 — applying the hooks and generating

hooks = [(f"blocks.{layer}.hook_resid_post", make_hook(layer)) for layer in target_layers]

with bridge.hooks(fwd_hooks=hooks):
    forced_start = "Sure! Here are a few reasons:"
    ans = bridge.generate(prompt + forced_start, max_new_tokens=150)
  • [ (...) for layer in target_layers ] is a list comprehension. A single-line loop that builds a list of (hook_point, hook_fn) pairs for each target layer.
  • with bridge.hooks(...): is a context manager. Hooks are enabled inside this block and automatically removed when exiting the block. No manual cleanup needed.
  • prompt + forced_start is a prefill text technique. We append a complying starting phrase to the prompt. The model sees that it has already "started complying" and naturally continues. This acts as a secondary steering mechanism.
